# 🔧 Panduan Setup MockAPI - Lengkap dengan Contoh Data

## 📋 Endpoint yang Harus Dibuat

Anda **HANYA** perlu membuat **1 endpoint** di MockAPI:

### Endpoint: `/movies`

Endpoint ini akan otomatis menyediakan semua operasi CRUD:
- `GET /movies` - Mendapatkan semua movies
- `GET /movies/:id` - Mendapatkan single movie
- `POST /movies` - Menambah movie baru
- `PUT /movies/:id` - Update movie (full)
- `PATCH /movies/:id` - Update movie (partial)
- `DELETE /movies/:id` - Hapus movie

MockAPI otomatis menyediakan semua method ini setelah Anda membuat resource.

---

## 🏗️ Struktur Field/Schema

Buat resource `movies` dengan field-field berikut:

| Field Name     | Type    | Description                              | Required | Example                        |
|----------------|---------|------------------------------------------|----------|--------------------------------|
| `id`           | string  | Auto-generated by MockAPI                | ✅ Yes   | "1", "2", "3"                  |
| `title`        | string  | Nama/judul film                          | ✅ Yes   | "Spider-Man"                   |
| `image`        | string  | URL/path gambar poster                   | ✅ Yes   | "/images/poster1.png"          |
| `alt`          | string  | Alt text untuk gambar                    | ✅ Yes   | "Spider-Man Poster"            |
| `hasNewEpisode`| boolean | Apakah ada episode baru                  | ✅ Yes   | true / false                   |
| `isTop10`      | boolean | Apakah termasuk Top 10                   | ✅ Yes   | true / false                   |
| `inMyList`     | boolean | Apakah ada di daftar user                | ✅ Yes   | true / false                   |

## 📝 Langkah-langkah Setup MockAPI

### Step 1: Buat Akun & Project

1. Buka https://mockapi.io/
2. Klik **"Sign Up"** atau **"Get Started"**
3. Buat akun dengan Google/Email
4. Setelah login, klik **"+ New Project"**
5. Beri nama project (contoh: "ChillStreamingAPI")

### Step 2: Buat Resource

1. Di dalam project, klik **"+ New Resource"**
2. Beri nama resource: `movies` (lowercase, tanpa spasi)
3. MockAPI akan auto-generate endpoint: `/api/v1/movies`

### Step 3: Tambahkan Fields

Klik **"Add Field"** dan tambahkan field satu per satu:

#### Field 1: title
- Name: `title`
- Type: `String`
- Klik "Save"

#### Field 2: image
- Name: `image`
- Type: `String`
- Klik "Save"

#### Field 3: alt
- Name: `alt`
- Type: `String`
- Klik "Save"

#### Field 4: hasNewEpisode
- Name: `hasNewEpisode`
- Type: `Boolean`
- Klik "Save"

#### Field 5: isTop10
- Name: `isTop10`
- Type: `Boolean`
- Klik "Save"

#### Field 6: inMyList
- Name: `inMyList`
- Type: `Boolean`
- Klik "Save"

### Step 4: Generate atau Input Data

Anda punya 2 pilihan:

**Opsi A: Auto-Generate (Quick)**
- Klik tombol "Generate Data"
- Pilih jumlah records (misalnya 10-15)
- MockAPI akan generate data random

**Opsi B: Input Manual (Recommended)**
- Klik "+" untuk add data baru
- Input data sesuai contoh di bawah

## 🔗 Copy Endpoint URL

Setelah selesai membuat resource dan input data:

1. Di MockAPI dashboard, lihat bagian **"Endpoint"**
2. Copy full URL (contoh: `https://6789abcd123456.mockapi.io/api/v1`)
3. Paste URL ini ke file `.env` di project Anda:

```env
VITE_API_BASE_URL=https://6789abcd123456.mockapi.io/api/v1
```

**⚠️ PENTING:** 
- Ganti `6789abcd123456` dengan ID MockAPI Anda yang sebenarnya
- Jangan lupa **RESTART development server** setelah update `.env`

## 🧪 Test Endpoint di MockAPI

MockAPI menyediakan built-in testing tool:

1. Di dashboard MockAPI, klik resource `movies`
2. Klik tab **"Try It"**
3. Pilih method (GET, POST, PUT, DELETE)
4. Klik **"Send Request"**
5. Lihat response

Contoh test:

### Test GET All Movies
```
Method: GET
URL: https://your-id.mockapi.io/api/v1/movies
```

Response akan berupa array semua movies.

### Test GET Single Movie
```
Method: GET
URL: https://your-id.mockapi.io/api/v1/movies/1
```

Response akan berupa single movie object.

### Test POST (Add Movie)

## 📊 Penjelasan Field `inMyList`

Field `inMyList` sangat penting untuk logika aplikasi:

- **`inMyList: true`** = Movie ada di "Daftar Saya" user
- **`inMyList: false`** = Movie hanya ada di katalog/available movies

Dalam contoh data di atas:
- Movie 1-3 memiliki `inMyList: true` (akan muncul di Daftar Saya)
- Movie 4-15 memiliki `inMyList: false` (hanya di katalog)

Saat user menambahkan movie:
1. Frontend akan POST movie baru dengan `inMyList: true`
2. Atau PATCH movie existing untuk set `inMyList: true`

## ❓ FAQ

**Q: Apakah saya perlu membuat endpoint terpisah untuk myList?**  
A: Tidak. Cukup 1 endpoint `/movies`. Aplikasi akan filter berdasarkan field `inMyList`.

**Q: Berapa banyak data yang harus saya input?**  
A: Minimal 10-15 data untuk testing. Anda bisa tambah lebih banyak sesuai kebutuhan.

**Q: Apakah image path harus real URL?**  
A: Tidak harus. Anda bisa gunakan path relatif seperti `/images/poster.png`. Aplikasi akan menggunakan local images.

**Q: Bagaimana jika saya tidak punya gambar poster?**  
A: Gunakan placeholder atau path yang sama untuk sementara. Yang penting struktur data-nya benar.

**Q: Apakah MockAPI gratis?**  
A: Ya, MockAPI memiliki free tier yang cukup untuk development dan testing.
